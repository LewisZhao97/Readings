## Comparison axes (what a future comparison page would structure around)

The cross-compare is essentially: **same problem (halve render load, synthesize intermediate frames), different platforms (mobile vs PC), different who-does-what (developer vs runtime), different motion-vector source (rasterizer vs video encoder).** Useful axes once ingested:

- Who generates motion vectors: app rasterizer pass (AppSW) vs runtime GPU video encoder block-matching (ASW).
- Developer opt-in vs always-on.
- 3DoF vs 6DoF depth handling — ASW 1.0 was 3DoF; ASW 2.0 and AppSW both use PTW for 6DoF.
- Frame cadence: Rift 90→45 (ASW) vs Quest 72/90/120 → 36/45/60 (AppSW).
- Artifact characteristics: video-encoder block artifacts (ASW) vs rasterizer-accurate per-pixel but with transparency/particle caveats (AppSW).
- Ecosystem: ASW is transparent to developers; AppSW requires OpenXR + Vulkan + modified shaders + URP/Unreal integration.
